# 012: ポインタ

## このレッスンで学ぶこと

- ポインタとは何か（メモリアドレス）
- `&`（アドレス取得）と `*`（デリファレンス）
- 値渡しと参照渡しの違い
- ポインタを使う場面・使わない場面
- `nil` ポインタ

## コード解説

### ポインタの基本

```go
x := 42
p := &x    // p は x のアドレスを持つ（型: *int）
fmt.Println(*p) // 42（デリファレンス: アドレスから値を取得）
*p = 100
fmt.Println(x)  // 100（p 経由で x が変わった）
```

- `&` → 変数のアドレスを取得
- `*` → アドレスから値を取り出す（デリファレンス）

### 値渡し vs ポインタ渡し

```go
// 値渡し: コピーが渡される → 元は変わらない
func double(n int) {
    n = n * 2 // 元の変数には影響しない
}

// ポインタ渡し: アドレスが渡される → 元が変わる
func doublePtr(n *int) {
    *n = *n * 2 // 元の変数が変わる
}
```

Go は**すべて値渡し**。ポインタを渡すと「アドレスの値」が渡される。

### いつポインタを使うか

| 場面 | ポインタ? | 理由 |
|------|----------|------|
| 構造体のフィールドを変更したい | Yes | メソッドのポインタレシーバ `(s *Struct)` |
| 大きな構造体を関数に渡す | Yes | コピーコストを避ける |
| `nil` を使って「値なし」を表現したい | Yes | ポインタの nil = 値が存在しない |
| 小さい値（int, bool, string） | No | コピーで十分。ポインタの方が遅い場合も |
| スライス、マップ | No | 内部的に参照型なので不要 |

### nil ポインタ

```go
var p *int       // ゼロ値は nil
fmt.Println(p)   // <nil>
// *p → パニック! nil デリファレンスは実行時エラー
```

ポインタを使う前に `nil` チェックが必要。

## Django 経験者向け

Python はすべてオブジェクト参照なので、値渡し/参照渡しを意識しない。
Go ではこの区別を常に意識する必要がある。
