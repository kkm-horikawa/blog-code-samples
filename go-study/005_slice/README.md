# 005: スライス

## このレッスンで学ぶこと

- 配列とスライスの違い
- スライスの作成方法（リテラル, `make`）
- `append` で要素追加
- スライシング（部分取得）
- `len` と `cap`

## コード解説

### 配列 vs スライス

```go
// 配列: サイズ固定（あまり使わない）
arr := [3]int{1, 2, 3}

// スライス: サイズ可変（こっちをよく使う）
slice := []int{1, 2, 3}
```

配列はサイズが型の一部（`[3]int` と `[5]int` は別の型）。
実用上は**ほぼスライスしか使わない**。

### make で作る

```go
s := make([]int, 5)      // 長さ5, ゼロ値で初期化
s := make([]int, 0, 10)  // 長さ0, 容量10（メモリ予約）
```

### append

```go
s := []int{1, 2}
s = append(s, 3)       // [1, 2, 3]
s = append(s, 4, 5, 6) // [1, 2, 3, 4, 5, 6]
```

`append` は**新しいスライスを返す**ので、必ず代入が必要。

### スライシング

```go
s := []int{0, 1, 2, 3, 4}
s[1:3]  // [1, 2]    （1番目から3番目の手前まで）
s[:3]   // [0, 1, 2]
s[2:]   // [2, 3, 4]
```

Python のスライスと同じ感覚。

## 注意点

- スライスは参照型。関数に渡すと元データを変更できる
- `append` で容量を超えると、内部で新しい配列が確保される

## やってみよう

1. 1〜10のスライスを作り、偶数だけの新しいスライスを作ってみよう
2. 文字列のスライスをソートしてみよう（`sort.Strings` を使う）
3. `len` と `cap` の違いを `make([]int, 3, 10)` で確認してみよう
