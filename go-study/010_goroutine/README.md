# 010: ゴルーチンとチャネル

## このレッスンで学ぶこと

- ゴルーチンの起動（`go` キーワード）
- チャネル（`chan`）でデータを送受信
- バッファ付きチャネル
- `select` で複数チャネルを待つ
- `sync.WaitGroup` で完了を待つ

## コード解説

### ゴルーチン

```go
go func() {
    fmt.Println("並行実行!")
}()
```

`go` を付けるだけで関数が**並行実行**される。OSスレッドより遥かに軽量。

### チャネル

```go
ch := make(chan string)     // チャネルの作成
ch <- "hello"               // 送信（ブロックする）
msg := <-ch                 // 受信（ブロックする）
```

チャネルはゴルーチン間の**通信パイプ**。
送信側と受信側が揃うまでブロックする（同期的）。

### バッファ付きチャネル

```go
ch := make(chan int, 3) // 3つまでバッファ
ch <- 1 // ブロックしない（バッファに余裕がある）
ch <- 2
ch <- 3
ch <- 4 // ここでブロック（バッファが満杯）
```

### select

```go
select {
case msg := <-ch1:
    fmt.Println("ch1:", msg)
case msg := <-ch2:
    fmt.Println("ch2:", msg)
case <-time.After(3 * time.Second):
    fmt.Println("タイムアウト")
}
```

複数のチャネル操作を同時に待てる。

### WaitGroup

```go
var wg sync.WaitGroup
wg.Add(3)        // 3つのゴルーチンを待つ
go func() {
    defer wg.Done() // 完了を通知
    // 処理
}()
wg.Wait()         // 全部終わるまでブロック
```

## Go の並行処理の格言

> "共有メモリで通信するな。通信でメモリを共有せよ。"

ロック（`sync.Mutex`）ではなく、チャネルでデータをやり取りするのが Go流。
